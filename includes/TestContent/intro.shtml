HTML / CSS Guidelines

HTML bootstrap with guides and good practices

The purpose of this document is to provide a visual guide to a basic set of guidelines for building browser-based application interfaces.

This document is meant to be used in conjunction with the Graphic Design Guidelines and the Good Practices Guidelines

It is intended to provide a framework for building structurally correct documents with minimal use of classes for styling. An application built with the basic CSS from this will position and space elements correctly.

There are many other guides available as well as tools to aid us such as color contrast algorithims, icon builders, color palette generators, motion (animation) generators etc. See list at end of document.

Principles

(In no particular order)

  • standards compliancy (W3C, WCAG)
  • inclusive - build for everyone
  • semantics - semantic implementation of code
  • progressive Enhancement - make sure it works everywhere before applying visual design / special effects (animations etc.). Link
  • modular - both HTML(Atomic principles) and CSS (SASS) for ease of finding / sharing / maintaining
  • agnosticism - be platform/browser independant
  • open - open for enhancement, amendment, evolution
  • classless - avoid speccifing classes at initial stage of projects
